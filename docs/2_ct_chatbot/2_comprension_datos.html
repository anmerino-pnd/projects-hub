📊 Comprensión de Datos para el Chatbot

Este documento describe los procesos de comprensión, normalización y análisis de datos que permiten al chatbot CT interactuar correctamente con los productos y servicios.

1. Embeddings Vectoriales

Los embeddings son representaciones vectoriales de texto que capturan el significado semántico.

1.1. Concepto

Un embedding es un vector denso de alta dimensión (típicamente 384 dimensiones para modelos como BERT) que representa un documento o frase de texto.

Nota: Los embeddings permiten que operaciones matemáticas sobre vectores reflejen relaciones semánticas entre documentos.

1.2. Ejemplo de Embedding

Para un documento de 1000 palabras, el embedding sería:

[0.234, -0.567, 0.891, ..., 0.123]

Cada elemento del vector representa una dimensión donde el valor indica la fuerza de una característica específica.

2. Tensores de Embeddings

Los tensores de embeddings organizan múltiples embeddings en una estructura bidimensional o multidimensional.

2.1. Estructura del Tensor

Un tensor de embeddings tiene la forma:

$$E = \begin{bmatrix} \vec{e}_1 \\ \vec{e}_2 \\ \vdots \\ \vec{e}_n \end{bmatrix}$$

Donde:

  • n = número de documentos
  • d = número de dimensiones (384 para BERT)

2.2. Ejemplo Numérico

Tensor de 3 documentos x 384 dimensiones:
┌─────────────────────────────────────────────────────────────────┐
│ Documentos: 3, Dimensiones: 384                                │
│                                                                 │
│ Doc 1: [0.12, -0.34, 0.56, ..., 0.78]                          │
│ Doc 2: [-0.45, 0.67, -0.89, ..., 0.23]                          │
│ Doc 3: [0.91, 0.12, -0.34, ..., 0.56]                          │
└─────────────────────────────────────────────────────────────────┘

3. Normalización de Datos

La normalización es crucial para garantizar que los embeddings sean comparables entre sí.

3.1. Normalización L2

La normalización L2 convierte cada vector a una longitud de 1:

$$\vec{v}_{norm} = \frac{\vec{v}}{\|\vec{v}\|} = \frac{\vec{v}}{\sqrt{\sum_{i=1}^{d} v_i^2}}$$

3.2. Ejemplo de Normalización

Vector original: [3, 4, 0]

Norma L2: √(3² + 4² + 0²) = √(9 + 16) = √25 = 5

Vector normalizado: [3/5, 4/5, 0] = [0.6, 0.8, 0]

3.3. Importancia

Importante: La normalización permite usar la similaridad coseno para comparar documentos, ya que todos los vectores tienen la misma longitud.

4. Análisis de Datos

El análisis de embeddings permite identificar relaciones semánticas entre documentos.

4.1. Similaridad Coseno

La similaridad coseno mide el ángulo entre dos vectores:

$$\cos(\theta) = \frac{\vec{a} \cdot \vec{b}}{\|\vec{a}\| \|\vec{b}\|} = \frac{\sum_{i=1}^{d} a_i b_i}{\|\vec{a}\| \|\vec{b}\|}$$

4.2. Interpretación

  • 1.0 = Vectores idénticos o paralelos
  • 0 = Vectores ortogonales
  • -1.0 = Vectores opuestos

4.3. Código de Ejemplo

import numpy as np
from sklearn.metrics.pairwise import cosine_similarity

# Dos embeddings
doc1 = np.array([0.6, 0.8, 0])
doc2 = np.array([0.7, 0.7, 0])

# Calcular similaridad
similarity = cosine_similarity([doc1], [doc2])[0][0]
print(f"Similaridad coseno: {similarity:.4f}")

4.4. Aplicaciones

  • Recomendación de productos similares
  • Clasificación de documentos
  • Búsqueda semántica
  • Grupos de productos relacionados
